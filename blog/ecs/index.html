ECS Suggestions

Components

1. Core Components

These are fundamental components that most entities will use.

  1. Transform: Position, rotation, and scale in 3D space.
  2. Velocity: Linear velocity for movement.
  3. Acceleration: Rate of change of velocity.
  4. AngularVelocity: Rotational speed.
  5. AngularAcceleration: Rate of change of angular velocity.
  6. Parent: Entity that this entity is a child of.
  7. Children: List of child entities.
  8. Tag: A string or enum for categorizing entities.
  9. Name: A human-readable identifier for the entity.
  10. Active: Whether the entity is active or inactive.

2. Rendering Components

Components related to visual representation.

  1. Mesh: Reference to a 3D mesh.
  2. Material: Reference to a material/shader.
  3. Texture: Reference to a texture.
  4. Color: Base color for rendering.
  5. UVOffset: Texture coordinate offset.
  6. Animation: Current animation state.
  7. Sprite: 2D image for rendering.
  8. Billboard: Always faces the camera.
  9. Light: Light source properties (color, intensity, range).
  10. ShadowCaster: Whether the entity casts shadows.
  11. ShadowReceiver: Whether the entity receives shadows.
  12. ParticleEmitter: Emits particles for effects.
  13. Decal: Projects a texture onto surfaces.
  14. LOD: Level of detail for rendering.
  15. Transparency: Controls opacity.
  16. Reflection: Reflectivity properties.
  17. Refraction: Refraction properties.
  18. Glow: Emissive glow effect.
  19. Outline: Outline rendering for selection or highlighting.
  20. Camera: Defines a viewport and projection.

3. Physics Components

Components for physics simulation.

  1. Collider: Defines collision shape (e.g., box, sphere, capsule).
  2. Rigidbody: Physical properties (mass, drag, gravity).
  3. Trigger: Detects overlaps without physical collision.
  4. Force: Applied force for movement.
  5. Torque: Applied rotational force.
  6. Gravity: Custom gravity settings.
  7. Friction: Surface friction.
  8. Bounciness: How much the entity bounces on collisions.
  9. Kinematic: Whether the entity is controlled by scripts rather than physics.
  10. RaycastHit: Stores results of a raycast.
  11. Buoyancy: Simulates floating in fluids.
  12. WindAffected: Responds to wind forces.
  13. HingeJoint: Rotational constraint between entities.
  14. SpringJoint: Elastic constraint between entities.
  15. FixedJoint: Rigidly connects two entities.
  16. Ragdoll: Simulates physics-based character animation.
  17. WaterSurface: Defines a water plane for physics interactions.
  18. Cloth: Simulates cloth physics.
  19. Softbody: Simulates soft, deformable physics.
  20. ExplosionForce: Applies force in a radial pattern.

4. Animation Components

Components for animating entities.

  1. Animator: Controls animation states and transitions.
  2. BlendShape: Morph target for facial expressions.
  3. Bone: Part of a skeleton for rigging.
  4. IKTarget: Target for inverse kinematics.
  5. AnimationClip: Reference to a specific animation.
  6. AnimationSpeed: Playback speed of animations.
  7. AnimationEvent: Triggers events during animation playback.
  8. LipSync: Synchronizes mouth movements with audio.
  9. FacialExpression: Controls facial animations.
  10. Gesture: Hand or body gestures for characters.

5. Audio Components

Components for sound and music.

  1. AudioSource: Plays sounds or music.
  2. AudioListener: Represents the “ear” for 3D audio.
  3. SoundEffect: Short audio clip for effects.
  4. MusicTrack: Background music.
  5. Volume: Controls audio volume.
  6. Pitch: Controls audio pitch.
  7. SpatialBlend: Balances 2D and 3D audio.
  8. ReverbZone: Applies reverb effects.
  9. AudioClip: Reference to an audio file.
  10. AudioLoop: Whether the audio loops.

6. AI and Behavior Components

Components for AI and entity behavior.

  1. AIState: Current state of the AI (e.g., idle, chasing).
  2. Pathfinding: Pathfinding data and goals.
  3. Waypoint: A point in a path.
  4. BehaviorTree: Hierarchical AI decision-making.
  5. Blackboard: Shared data for AI decisions.
  6. Target: Entity to target (e.g., for attacks or movement).
  7. VisionCone: Defines what the entity can “see.”
  8. HearingRange: Defines what the entity can “hear.”
  9. PatrolRoute: Path for patrolling behavior.
  10. Flocking: Group movement behavior.
  11. Steering: Steering behaviors (e.g., seek, flee).
  12. Aggression: How aggressive the entity is.
  13. Fear: How fearful the entity is.
  14. Curiosity: How curious the entity is.
  15. Memory: Stores information about the environment.
  16. DecisionMaker: Makes decisions based on inputs.
  17. UtilityAI: Scores actions based on utility.
  18. Goal: What the AI is trying to achieve.
  19. Action: An action the AI can perform.
  20. ReactionTime: How quickly the AI reacts.

7. Gameplay Components

Components for gameplay mechanics.

  1. Health: Current and maximum health.
  2. Damage: Amount of damage the entity can deal.
  3. Armor: Reduces incoming damage.
  4. Experience: Tracks experience points.
  5. Level: Current level of the entity.
  6. Inventory: Stores items.
  7. Equipment: Items currently equipped.
  8. Weapon: Defines weapon properties.

  14. Quest: Tracks quest progress.
  15. Dialogue: Stores dialogue lines.

  17. Pickup: Can be picked up by the player.
  18. Door: Can be opened or closed.
  19. Switch: Can be toggled on or off.
  20. Key: Unlocks something (e.g., a door).
  21. Lock: Requires a key to unlock.

  23. Checkpoint: Saves progress.
  24. Respawn: Defines respawn behavior.
  25. Team: Which team the entity belongs to.
  26. Faction: Relationship with other factions.
  27. Reputation: How others perceive the entity.
  28. Stealth: How stealthy the entity is.
  29. Detection: How easily the entity is detected.
  30. Reward: What the entity gives when defeated or interacted with.

9. Networking Components

Systems

Here’s a list of 200 systems for an ECS architecture, categorized by functionality and referencing the components from the previous list. Systems are designed to process components logically, ensuring efficiency and scalability while avoiding anti-patterns.

1. Core Systems

Process foundational components like transforms, hierarchy, and basic state.

  1. MovementSystem: Updates Transform using Velocity and Acceleration.
  2. RotationSystem: Updates rotation using AngularVelocity and AngularAcceleration.
  3. ParentingSystem: Updates child entities based on Parent and Children.
  4. TagSystem: Handles entity categorization via Tag.
  5. NamingSystem: Manages entity Name for debugging/UI.
  6. ActivationSystem: Toggles entity visibility/behavior via Active.
  7. HierarchySystem: Maintains parent-child relationships.
  8. BoundsSystem: Calculates bounding volumes for Transform and Collider.
  9. LifetimeSystem: Tracks entity lifespan (e.g., for temporary effects).
  10. SpawnSystem: Instantiates entities with predefined components.

2. Rendering Systems

Handle rendering, lighting, particles, and visual effects. 11. RenderMeshSystem: Draws entities with Mesh, Material, and Transform. 12. SpriteRenderingSystem: Renders 2D Sprite entities. 13. LightingSystem: Updates Light sources and shadows. 14. ShadowCastingSystem: Manages ShadowCaster and ShadowReceiver. 15. ParticleUpdateSystem: Simulates ParticleEmitter effects. 16. DecalSystem: Projects Decal textures onto surfaces. 17. TransparencySystem: Blends Transparency for translucent objects. 18. OutlineSystem: Renders Outline effects for highlighted entities. 19. CameraSystem: Updates Camera view/projection matrices. 20. LODSystem: Swaps LOD meshes based on distance. 21. ReflectionSystem: Renders Reflection probes. 22. RefractionSystem: Handles Refraction effects. 23. GlowSystem: Applies Glow emissive effects. 24. BillboardSystem: Orients Billboard entities toward the camera. 25. UISystem: Renders UIElement, Button, Text, etc. 26. HUDSystem: Updates HealthBar, Minimap, and Scoreboard. 27. WeatherSystem: Renders Rain, Snow, and Fog. 28. PostProcessingSystem: Applies screen-space effects (e.g., blur, color grading). 29. SkyboxSystem: Renders the skybox or DayNightCycle. 30. VFXSystem: Manages visual effects like Fire, Smoke, and Explosion.
